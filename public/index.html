Why this exists

The x402 standard (Coinbase, May 2025) lets AI agents pay each other USDC over HTTP. 169+ SDKs. 13,000+ registered services. But x402 has no identity, no policy, no kill switch. A compromised or misaligned agent can drain its owner's wallet. The owner has no way to stop it.

Last month, researchers published Five Attacks on x402 (arXiv:2605.11781) — grant-before-settle, replay, missing resource binding, fire-and-forget settlement, missing cache-control. Every x402 SDK ships with these holes.

ERC-8004 ("Trustless Agents") shipped on Base in Feb 2026, with 17,600+ agents already registered. It has identity + reputation. PayGate is the missing operational layer that uses them.

What PayGate adds

🔐 ERC-8004 identity

Each agent gets a portable on-chain handle on Base. Other agents can verify who they're paying before they pay.

📊 On-chain policy

Per-call cap, per-epoch cap, allowlist. The human sets it once, the chain enforces it forever — not the SDK.

🛑 Kill switch

One transaction pauses the agent. No race condition, no admin key, no "please contact support".

⭐ Reputation

After every call, feedback is written to the ERC-8004 Reputation Registry. Other agents see it before they pay.

Drop-in usage

import { wrap, call, register } from "@paygate/sdk";

// 1. Register the agent (one-time)
const { paygateAgentId, policyAddress } = await register(cfg, {
  perCallLimit: 100_000n,   // $0.10
  perEpochLimit: 1_000_000n, // $1.00
  epochDuration: 86400,      // 24h
});

// 2. Wrap any async function as a PayGate-protected agent
const sentimentAgent = wrap(cfg, async (input) => {
  return { sentiment: "positive" };
}, { priceUSDC: 10000n }); // $0.01 per call

// 3. Another agent can call it
const result = await call(cfg, "https://sentiment.example.com/", {
  amount: 10000n,
  body: { text: "PayGate is amazing" },
});
